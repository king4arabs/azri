# AZRI iOS + watchOS

Patient iPhone app + Apple Watch companion. Arabic-first (RTL), HealthKit
read scopes, watchOS app for on-wrist alerts and one-tap episode logging.

## Layout

```
apps/ios/
├── Package.swift             ← Swift Package Manager root (shared module)
├── Azri/                     ← iPhone app target
│   ├── AzriApp.swift
│   ├── Info.plist
│   ├── ContentView.swift
│   └── Features/
│       ├── EpisodeLogView.swift
│       ├── TrendsView.swift
│       └── AlertRelayView.swift
├── AzriWatch/                ← watchOS app target
│   ├── AzriWatchApp.swift
│   ├── Info.plist
│   ├── ContentView.swift
│   └── Features/
│       ├── OnWristAlertView.swift
│       └── QuickEpisodeView.swift
├── AzriShared/               ← shared logic (HealthKit, ingest, models)
│   ├── HealthKit/
│   │   ├── HealthKitAuthorizer.swift
│   │   └── BiosignalCollector.swift
│   ├── Ingest/
│   │   ├── IngestClient.swift
│   │   └── IngestQueue.swift
│   ├── Models/
│   │   └── ContractTypes.swift   ← Swift mirror of @azri/contracts
│   └── Connectivity/
│       └── WatchConnectivityBridge.swift
├── AzriTests/
│   └── HealthKitAuthorizerTests.swift
└── AzriUITests/
    └── EpisodeLogUITests.swift
```

## Why this layout

- `AzriShared/` is pure Swift, no UIKit / SwiftUI imports. iPhone and
  Watch targets both depend on it — keeps biosignal collection and the
  ingest queue testable in isolation.
- `Models/ContractTypes.swift` is a hand-maintained Swift mirror of the
  `@azri/contracts` Zod schemas. When contracts ship a MAJOR bump, this
  file MUST be updated in the same PR.
- `WatchConnectivityBridge.swift` is the only place WatchConnectivity is
  imported, so the messaging contract between phone and watch is one
  reviewable file.

## Build

This scaffold ships as Swift source plus a `Package.swift`. Generate an
Xcode workspace:

```bash
cd apps/ios
swift package generate-xcodeproj   # legacy; or:
xed .                               # open the SwiftPM root in Xcode 16+
```

The Xcode project for the iPhone and Watch targets is generated by
`xcodegen` from `project.yml` (see file in this folder). Add the two app
targets in Xcode if regenerating from scratch — see
`SDK_DOCUMENTATION.md`.

## HealthKit usage strings (required by Apple)

Both `Info.plist` files declare:

| Key | Use |
| --- | --- |
| `NSHealthShareUsageDescription` | "AZRI uses your heart rate, HRV, sleep, and movement data to support epilepsy monitoring and alert your trusted caregivers." |
| `NSHealthUpdateUsageDescription` | "AZRI does not write to HealthKit at this time." |
| `NSMotionUsageDescription` | "AZRI uses motion data on your Apple Watch to detect possible falls and unusual activity patterns." |

Bilingual `InfoPlist.strings` files exist under `Azri/ar.lproj/` and
`Azri/en.lproj/` so the prompts present in the patient's language.

## Tests

```bash
swift test
```

`HealthKitAuthorizerTests` covers permission-request flow with a fake
authoriser; UI tests cover the episode log RTL flip.
