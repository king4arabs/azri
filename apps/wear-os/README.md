# AZRI Wear OS

Kotlin + Jetpack Compose Wear OS app for Samsung Galaxy Watch (Wear OS 4+)
and other Wear OS watches (Pixel Watch, etc.). Uses Health Services for
high-frequency biosignals; foreground service keeps sensor capture alive
while the screen is off.

> **Samsung note.** Modern Galaxy Watches (Watch 4 onward) run Wear OS;
> older watches (Galaxy Watch 3 and earlier) run Tizen and require a
> companion app on the phone. AZRI's Tizen story is documented in
> `SDK_DOCUMENTATION.md` under "Wearables → Samsung legacy"; the Wear OS
> path is the actively-maintained one.

## Build

```bash
cd apps/wear-os
./gradlew :app:assembleDebug
./gradlew :app:test
```

Android Studio Hedgehog (or newer) opens the project directly.

## Permissions requested

The Wear OS Health Services API gates each data source on a runtime
permission. The watch app requests:

| Permission | Why |
| --- | --- |
| `android.permission.BODY_SENSORS` | Heart rate |
| `android.permission.BODY_SENSORS_BACKGROUND` | Continue HR while screen off |
| `android.permission.ACTIVITY_RECOGNITION` | Step count |
| `android.permission.POST_NOTIFICATIONS` | Foreground service notification + alerts |

Bilingual strings are split between `values/strings.xml` (English) and
`values-ar/strings.xml` (Arabic). The manifest declares
`android:supportsRtl="true"`.

## Tizen / older Samsung watches

Galaxy Watch 3 and earlier run Tizen, which Samsung no longer ships
new SDK updates for. The recommended posture for AZRI is:

1. Display a polite in-app message on the phone explaining that the
   watch model is unsupported and recommending a Wear OS companion.
2. Ingest signals from the phone's HealthKit/Health Connect bridge in
   the meantime — degraded coverage is better than none.
3. Document the model list and bridge fallback in `OPERATIONS.md`.
